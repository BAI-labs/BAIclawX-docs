# DingTalk

## Prerequisites

- DingTalk account

## Configuration Steps

### 1. Create an Organization

1. Click **Create or Join Organization**

![](assets/Pasted%20image%2020260319004613.png)

2. Click "Organization", then select **Create Organization/Team**

![](assets/Pasted%20image%2020260319005014.png)

3. Fill in the information and create (the "Organization Name" needs to be more than three characters, other content can be filled arbitrarily)

![](assets/Pasted%20image%2020260319005321.png)

### 2. Create a Robot

Go to [DingTalk Developer Platform](https://open-dev.dingtalk.com), select the organization you just created.

![](assets/Pasted%20image%2020260319005820.png)

1. Select **Create**

![](assets/Pasted%20image%2020260319010139.png)

2. In the left sidebar, select "Robot" and click **Create Application**

![](assets/Pasted%20image%2020260319010213.png)

3. Fill in the "Application Name" and "Application Description" and click **Save**

![](assets/Pasted%20image%2020260319010557.png)

4. In the left sidebar, select **Add Application Capability**, and choose to add "Robot"

![](assets/Pasted%20image%2020260319010802.png)

5. Enable robot configuration and fill in the content

![](assets/Pasted%20image%2020260319010941.png)

![](assets/Pasted%20image%2020260319011156.png)

### 3. Adjust Permissions and Publish

1. In the left sidebar, select "Permission Management", search for "messages", check the two items in the image and click **Batch Apply**

![](assets/Pasted%20image%2020260319011512.png)

2. In the left sidebar, select "Version Management and Release", click **Create New Version** (must create version information to publish)

![](assets/Pasted%20image%2020260319011628.png)

3. Enter "Application Version Number" and "Version Description" and **Save** to complete publishing
4. Select "Credentials and Basic Information", copy **APP ID** and **Client Secret** to BAIclaw and save to connect

### 4. Search and Use the Robot

Search for the "Robot Name" you just created in the search box of the DingTalk **Messages** bar, and it will be displayed under the function bar.
